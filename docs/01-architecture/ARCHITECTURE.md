## 模块设计

系统划分为以下主要模块，每个模块具有明确的职责和接口，支持独立开发与部署：

| 模块 | 核心职责 | 关键服务/路由 |
|------|----------|---------------|
| **用户与权限管理 (RBAC)** | 租户隔离、用户认证、角色权限、审计日志 | `AuthMiddleware`, `RoleManager`, `/api/users`, `/api/roles` |
| **商品与库存管理** | SKU/SPU、库位、批次/效期、库存预占/释放 | `ProductConstraintService`, `StockAllocationService`, `/api/inventory` |
| **订单与波次管理** | 订单全生命周期、波次策略、分配算法 | `WorkOrderService`, `/api/orders`, `/api/wave-strategy` |
| **仓储作业** | 入库/上架、出库/拣货、盘点、补货 | `SortingService`, `VerificationService`, `PackingService`, `LoadingService`, `ReplenishmentScheduler` |
| **财务与计费** | 计费策略、阶梯定价、账单生成、对账 | `BillingEngine`, `/api/billing` |
| **报表与数据分析** | 多维分析、驾驶舱、导出、实时订阅 | `ReportService`, `/api/reports` |
| **设备与租户管理** | PDA/扫码枪注册、固件升级、租户配置 | `BlackboxReceivingService`, `/api/devices` |

## 数据流设计

### 核心业务流数据流

#### 入库流程
```
供应商送货 → 收货单创建 (API) → 质检任务 (异步) → 质检通过
    → 上架工单派发 (Workflow) → PDA 执行上架 → 库存写入 (RPC)
    → 事件发布 (库存变更) → 补货检查触发 → 缓存失效 (Edge)
```

#### 出库/拣货流程
```
订单下发 (API) → 波次规划 (定时/手动) → 库存分配锁定 (RPC 乐观锁)
    → 拣货工单生成 → PDA 拣货执行 → 复核/打包工单
    → 装车工单 → 发货出库 → 库存扣减 (RPC) → 计费触发
```

#### 补货流程
```
定时调度 (Cron/Event) → 识别低库存库位 (SQL/RPC)
    → 计算补货量 (算法服务) → 生成补货工单批次
    → PDA 执行移库 → 库存同步 → 缓存刷新
```

## 技术栈选型

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **运行时** | Node.js | 20 LTS | 长期支持、生态成熟 |
| **Web 框架** | Express | 4.19 | 简洁、中间件丰富、团队熟悉 |
| **类型系统** | TypeScript | 5.1 | 类型安全、重构友好 |
| **数据库** | PostgreSQL (Supabase) | 16 | RLS 原生多租户、Realtime、Edge Functions |
| **ORM/查询** | Supabase JS Client | 2.110 | 类型安全、实时订阅、Auth 集成 |
| **缓存/队列** | Redis | 7.2 | 高性能、支持流/锁/发布订阅 |
| **边缘计算** | Cloudflare Workers | - | 全球分布、零冷启动、KV 存储 |
| **API 网关** | Kong | 3.4 | 插件生态、限流、认证、可观测 |
| **容器编排** | Kubernetes | 1.28 | 生产标准、自愈、弹性伸缩 |
| **服务网格** | Istio | 1.20 | mTLS、流量管理、可观测 |
| **监控** | Prometheus + Grafana | 2.48 / 10.2 | 云原生标准、查询语言强 |
| **日志** | Loki + Promtail | 2.9 | 成本低、Label 索引、Grafana 集成 |
| **告警** | Alertmanager | 0.26 | 分组、抑制、静默、多渠道 |
| **CI/CD** | GitHub Actions | - | 原生集成、Matrix、可复用 Workflow |
| **测试** | Vitest + Playwright | 4.1 / 1.44 | 快速、类型安全、E2E 可靠 |
