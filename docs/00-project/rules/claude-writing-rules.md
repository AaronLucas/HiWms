# Claude 写入规则

> 所有文件修改必须遵循的执行机制规范。

---

## 1. Patch 模式
- 所有文件修改必须以 **unified diff patch** 的形式输出。
- 使用 `Edit` 工具（精确替换）或 `Write` 工具（全新文件），不得使用其他变通方式。

## 2. 自动应用 Patch
- Claude 必须自动应用 patch，不得要求用户手动应用。
- 工具调用即为应用动作，完成后直接报告结果。

## 3. 禁止覆盖写入
- 不得直接覆盖写入任何已存在文件（除非是全新创建）。
- 修改现有文件必须使用 `Edit` 工具进行精确替换。

## 4. 失败重试机制
- 如果 patch 应用失败（旧字符串不匹配、文件不存在等），Claude 必须：
  1. 重新读取目标文件当前内容
  2. 基于最新内容生成修正后的 patch
  3. 再次应用
- 连续失败 3 次后主动询问用户。

## 5. 单文件写入
- Claude 必须避免并发修改多个文件，每次只处理一个文件。
- 多文件变更按顺序串行执行，完成一个再处理下一个。

## 6. 文件占用检测
- Claude 必须避免在文件被其他进程占用时写入（如 tsserver、prettier、dev server 正在读取）。
- 检测到占用时等待 1-2 秒后重试，最多重试 3 次。

## 7. 禁止写入 CLAUDE.md
- 所有写入行为必须遵守本规则体系，不得直接写入根目录 `CLAUDE.md`。
- `CLAUDE.md` 仅作为引用入口，规则内容写入 `docs/00-project/rules/` 对应文件。
