# 上下文管理规则

> 定义 Claude Code 会话中上下文的加载、压缩、优先级与隔离机制。

---

## 1. 自动加载层级（优先级由高到低）

| 层级 | 内容 | 加载方式 | 生命周期 |
|------|------|----------|----------|
| **L0 核心规则** | `CLAUDE.md` | 启动时自动读取 | 全会话常驻 |
| **L1 执行规则** | `docs/00-project/rules/*.md` | 首次需要时懒加载 | 会话级缓存 |
| **L2 设计规范** | `docs/01-architecture/` `docs/02-api/` 等 | 任务涉及时显式读取 | 任务级 |
| **L3 项目知识** | `docs/00-project/ROADMAP.md` 等 | 任务涉及时显式读取 | 任务级 |
| **L4 代码上下文** | 相关源文件 | 任务执行时精准读取 | 调用级 |

---

## 2. 上下文压缩策略

### 2.1 Token 预算分配
- **总预算**：会话上下文窗口的 80%（留 20% 给模型推理）
- **L0-L1**：常驻，不参与压缩
- **L2-L4**：按任务相关性评分，低分优先压缩/丢弃

### 2.2 压缩动作
| 触发条件 | 动作 |
|----------|------|
| 剩余 Token < 30% | 摘要化 L2/L3（保留标题+关键结论，丢弃细节） |
| 剩余 Token < 15% | 仅保留 L0+L1+当前任务相关代码片段 |
| 剩余 Token < 5% | 强制结束当前任务，要求用户确认是否继续 |

### 2.3 摘要化规则
- 文档 → 保留「标题、核心结论、关键参数、版本号」
- 代码 → 保留「签名、关键逻辑行、类型定义」
- 对话历史 → 保留「决策点、确认事项、待办」

---

## 3. 隔离机制

### 3.1 任务级隔离
- 每个顶层任务（用户指令）维护独立的上下文切片
- 任务完成后自动释放 L2-L4 上下文，仅保留决策记录

### 3.2 子 Agent 隔离
- 子 Agent 拥有独立上下文窗口
- 父会话仅接收子 Agent 返回的结构化结果，不继承其完整上下文

### 3.3 文件级隔离
- 同一文件的多次读取在会话内缓存
- 检测到文件外部修改（mtime 变化）时自动失效缓存

---

## 4. 显式加载指令

用户可通过以下方式控制上下文：

| 指令 | 效果 |
|------|------|
| `加载 <文档路径>` | 强制读取指定文档到当前上下文 |
| `卸载 <文档路径>` | 从当前上下文移除指定文档缓存 |
| `清理上下文` | 触发一次全量压缩（保留 L0-L1） |
| `上下文状态` | 显示当前各层级占用估算 |
